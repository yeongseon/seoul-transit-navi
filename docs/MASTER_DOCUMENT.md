# ソウル交通ナビ 개발 문서 v0.1

> **MVP Scope Notice**: 현재 MVP는 **지하철 전용**으로 구현되었습니다. 버스, 공항철도, 출구 상세 안내 등은 향후 확장 예정입니다. 아래 문서는 전체 제품 비전을 포함하며, MVP에서 구현된 범위는 지하철 1~9호선 (111개 역)의 경로 검색, 일본어 역명 검색, 관광지 안내입니다.

## Current MVP Implementation
- Seoul subway lines 1-9 (~111 stations)
- Japanese/Korean bilingual UI
- Route search via ODsay API + graph fallback
- 15 curated tourist destinations
- Station search with multilingual aliases
- Shareable route URLs (90-day TTL)
- Airport-to-hotel guide
- Area guides (4 districts)
- Station guides (confusing stations)
- Transit tips page
- Subway map visualization

## 1. 프로젝트 개요

### 1.1 서비스명
- **ソウル交通ナビ**
- 서브카피: **ソウルの地下鉄を、日本語でわかりやすく。**

### 1.2 한 줄 정의
일본인 관광객이 서울에서 지하철과 버스를 **일본어로 쉽고 자세하게 이해할 수 있도록 돕는 모바일 퍼스트 웹서비스**.

### 1.3 제품 방향
이 서비스는 단순한 지도 복제 서비스가 아니다. 핵심은 아래 3가지다.
- **JP 스타일 정보 UX**: 정보량이 많아도 괜찮다. 대신 구조화되어야 한다.
- **관광객 중심 UX**: 현지인용이 아니라 서울이 익숙하지 않은 일본인 기준으로 설계한다.
- **앱 확장형 웹서비스**: 초기에는 모바일 웹/PWA로 시작하고, 이후 앱으로 전환 가능한 구조로 설계한다.

## 2. 목표와 비목표

### 2.1 목표
- 일본인 관광객이 서울 대중교통을 **불안 없이** 사용할 수 있게 한다.
- 경로 자체뿐 아니라 **환승, 출구, 주의사항, 버스 정보**까지 이해 가능하게 제공한다.
- 앱 설치 없이 링크 접속만으로 바로 사용할 수 있게 한다.
- 향후 앱, 여행 정보, 즐겨찾기, 오프라인 일부 기능으로 확장 가능하게 만든다.

### 2.2 비목표
- 한국인 현지인을 메인 타겟으로 하지 않는다.
- 초기에 한국 전역을 다루지 않는다. **서울/수도권 주요 관광 동선** 중심으로 시작한다.
- 실시간 대규모 지도 엔진을 직접 개발하지 않는다.
- 초기에 완전한 다국어 서비스를 만들지 않는다. **일본어 우선**이다.

## 3. 타겟 사용자

### 3.1 핵심 타겟
- 일본에서 한국 여행을 온 20~40대 관광객
- 혼자 여행 또는 2~3인 자유여행 사용자
- 한국어를 거의 모르거나, 영어보다 일본어 안내를 선호하는 사용자

### 3.2 대표 페르소나

#### Persona A: 첫 서울 여행자
- 도쿄 거주, 20대 후반 여성
- 명동, 성수, 홍대, 잠실 같은 관광지를 방문
- 지도앱은 쓰지만 한국 교통 시스템은 익숙하지 않음
- 원하는 것: “어디서 갈아타고 몇 번 출구로 나가는지”

#### Persona B: 부모 동반 여행자
- 30대 후반 부부
- 캐리어 있음
- 이동이 복잡하면 피하고 싶어함
- 원하는 것: “환승 적은 경로”, “버스가 더 편한지”, “계단이 많은지”

#### Persona C: 공항 진입 사용자
- 인천공항 도착 직후
- T-money, 공항철도, 공항버스가 헷갈림
- 원하는 것: “가장 쉬운 시내 이동 방법”

## 4. 문제 정의

일본인 관광객은 서울 대중교통을 이용할 때 아래 문제를 자주 겪는다.

1. **노선도 이해보다 목적지 이동이 중요하다.**
   - 사용자는 “2호선, 4호선”보다 “홍대에서 명동 어떻게 가지?”가 중요하다.

2. **환승 과정이 불안하다.**
   - 몇 정거장 후 어디서 갈아타는지, 어려운 환승인지 알기 어렵다.

3. **출구 정보가 중요하다.**
   - 목적지에 도착해도 몇 번 출구인지에 따라 실제 이동 난이도가 달라진다.

4. **버스는 더 어렵다.**
   - 지하철보다 버스 이용 진입 장벽이 훨씬 높다.

5. **일본 사용자에게는 상세 설명이 안심감을 준다.**
   - 정보가 적으면 오히려 불안해한다.

## 5. 핵심 가치 제안

### 5.1 제품 가치
- **일본어로 자세히 안내한다.**
- **최단 경로만이 아니라 쉬운 경로를 보여준다.**
- **관광객이 실제로 필요한 정보 중심으로 설명한다.**
- **서울 교통을 ‘이해 가능한 UX’로 바꾼다.**

### 5.2 핵심 메시지
- 처음 서울에 와도, 길을 잃지 않는다.
- 지하철도 버스도, 일본어로 쉽게 이해할 수 있다.
- 환승, 출구, 요금, 시간까지 한 번에 확인할 수 있다.

## 7.2 검색 자동완성

> **[MVP 현황]** 역명 검색과 관광지 검색을 모두 지원합니다. 관광지 선택 시 최인접 역으로 자동 연결됩니다.

### 요구사항
- 일본어 역명 검색 가능
- 한국어 역명 매핑
- 영어 표기 일부 지원
- 관광지 명칭 검색 가능
- 역/관광지/랜드마크 우선순위 제공

### 예시
- 明洞 → 명동역/명동 관광지역
- 弘大 → 홍대입구역
- 景福宮 → 경복궁역 또는 경복궁 주변

## 7.3 경로 결과 화면

### 요구사항
- 추천 경로 3개까지 제공
- 각 경로마다 아래 정보 노출
  - 소요 시간
  - 예상 요금
  - 환승 횟수
  - 교통수단 유형
  - 추천 이유

### 추천 라벨 예시
- 最速
- 乗り換えが少ない
- わかりやすい
- バス移動が少ない

### 일본식 설명 예시
- 初めての方にもわかりやすいルートです。
- 乗り換えは1回だけです。
- 地下鉄中心なので移動しやすいです。

---

## 7.4 상세 경로 안내

### 요구사항
각 경로에 대해 단계별로 아래 정보 제공
- 어느 역에서 승차하는지
- 몇 정거장 이동하는지
- 어느 역에서 하차하는지
- 어떤 노선으로 갈아타는지
- 버스라면 버스 번호와 승차 정류장
- 도보 구간 거리/시간

### 예시
- 弘大入口駅で2号線に乗ります。
- 6駅移動して忠正路駅で降ります。
- 4号線に乗り換えます。
- 4駅移動して明洞駅で降ります。
- 6番出口から出るとわかりやすいです。

## 10. UX 원칙

### 10.1 JP 스타일 UX 원칙
- 정보는 충분히 제공한다.
- 단, 구조화와 시각적 계층이 중요하다.
- 추천 이유를 같이 보여준다.
- 사용자의 불안을 줄이는 표현을 사용한다.
- 결과 하나만 강요하지 않고 선택지를 준다.

### 10.2 UI 원칙
- 모바일 우선
- 카드형 정보 구조
- 텍스트 + 아이콘 + 작은 설명 조합
- 결과는 스크롤 친화적
- 한 화면에서 핵심 정보가 바로 보이도록 설계

### 10.3 카피 원칙
- 짧고 명확한 일본어
- 지나치게 기계 번역 같은 표현 금지
- “안심”, “알기 쉬움”, “초보자도 가능” 같은 정서적 메시지 허용

## 12. 기술 아키텍처

- Next.js
- TypeScript
- Tailwind CSS

#### 백엔드
선택지 A: Cloudflare Workers 기반 BFF
선택지 B: Next.js API Routes + 외부 API/데이터 레이어

- **Frontend**: Next.js App Router
- **BFF/API Layer**: Cloudflare Workers 또는 Next server functions
- **Cache**: Cloudflare CDN / KV / Cache API
- **Data Store**: D1 또는 외부 PostgreSQL/Supabase

### 12.3 권장 구조
- 브라우저
- Cloudflare CDN
- Web frontend
- Edge API/BFF
- Route engine / data service

## 16. 라우팅/추천 로직 방향

### 16.1 기본 원칙
최단 시간만으로 추천하지 않는다.

### 16.2 추천 점수 요소
- 총 시간
- 환승 횟수
- 도보 거리
- 버스 포함 여부
- 관광객 난이도
- 출구 편의성
- 캐리어 이동 불편 여부

### 16.3 추천 라벨 생성
- 총 시간 짧음 → 最速
- 환승 적음 → 乗り換えが少ない
- 관광객 난이도 낮음 → わかりやすい
- 공항버스 포함 + 캐리어 친화 → 荷物が多い方向け

## 31. 최종 요약

**ソウル交通ナビ**는 일본인 관광객을 위한 서울 대중교통 안내 서비스다.
핵심은 “일본어 지원” 자체가 아니라, **일본인이 안심할 수 있는 정보 구조**다.

따라서 제품은 다음 원칙 위에서 개발되어야 한다.
- 정보는 충분히 제공한다.
- 관광객 기준으로 쉽게 설명한다.
- 지하철과 버스를 함께 다룬다.
- 모바일 웹으로 빠르게 시작한다.
- PWA와 앱 확장을 염두에 둔다.

이 방향이면 단순한 길찾기 도구가 아니라,
**일본인 관광객을 위한 서울 이동 UX 레이어**로 발전시킬 수 있다.
